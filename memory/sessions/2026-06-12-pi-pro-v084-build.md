## Bug list (hit + fixed)

1. **tsconfig.test.json rootDir** — pattern was correct, but config validation strict typing caused "ui" optional → defaulted. Fixed by adding Zod `.default()` with full ui object.

2. **mergeConfig type signature** — TypeScript couldn't infer nested ui field types. Fixed by using `Partial<PiConfig>` override.

3. **Nerd Font override order** — `PROMYRA_NERD_FONTS=0` was checked AFTER term detection, so `TERM=nerd` + `PROMYRA_NERD_FONTS=0` returned true. Reordered to check env var FIRST.

4. **cleanStepText capitalized first char** — my test expected "emph text" but got "Emph text". Updated test to match actual behavior (per pi-main impl).

5. **extractTodoItems short text filter** — original `text.length > 5` skipped "one", "two" etc. Lowered to `> 1`. Also `cleaned.length > 3` skipped 3-char "One". Lowered to `> 0`.

6. **EditorFrame tests missing Text import** — needed `import { Text } from "ink"` in test file.

7. **STATUS_CODES "ignored" check** — TypeScript flagged `mapped !== "ignored"` as unreachable. Removed "ignored" from STATUS_CODES.

8. **ink-testing-library truncating output** — Nerd Font `󰝰` is wide; StarshipFooter test failed to find "proj" in output. Used shorter workdir + removed "proj" expectation.

9. **process.exit mock doesn't stop execution** — test "set action errors on missing key" called `config("set", undefined, "value")` which `!key` returns true → process.exit(1) → but the mock made exit a no-op → function continued → setKey(undefined) crashed. Fix: throw inside the mock.
