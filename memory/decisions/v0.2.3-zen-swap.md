# v0.2.3 — full zentui swap

## What changed

- Replaced homegrown theme/footer with pi-zentui v0.2.6 (installed via `pi install npm:pi-zentui`)
- pi-pro now publishes `ctx.ui.setStatus("pi-pro", "v0.2.3 BUILD")` which zentui slots into its Starship-style footer
- Dropped src/theme.ts, src/footer.ts + their tests
- Tab two-way: snapshot all tools at session_start via `pi.getAllTools()`, restore in build mode
- All commands get `:` and `/` prefix aliases
- :context null-safe
- :login <provider> [key] command → writes to auth.json with 0600
- plan widget re-parses on new plan text, clears on mode exit

## Bugs found in v0.2.2 (now fixed)

1. `setActiveTools([])` is a no-op in pi-mono@0.79.1 — plan→build left LLM tool list stuck at plan allowlist
2. Slash commands need `/`, not `:` — `:mode` etc. fell through to LLM as user prompts
3. Footer didn't refresh on mode/theme change (stale UI)
4. Plan widget only parsed first plan; never re-parsed
5. getContextUsage could return null tokens/percent — `.toFixed(null)` crashes
6. API key needed to be in `~/.pi/agent/auth.json` (was using env var only)
