# v0.8.4 decisions

**Date:** 2026-06-12
**Build:** v0.8.4 "Workflow + UI Overhaul" (final +161 tests, 1321 total, 19 packages)
**Author:** OpenCode (Sid-approved)

## 1. Foundation first

v0.8.4 starts with `@pi/config` because every v0.9.0+ feature (plugins, LSP, web viewer, Telegram) reads from config. The package is small (34 tests) but enables a huge surface area later.

**Decision:** Build `@pi/config` first as commit 1. Other features depend on it.

## 2. Config schema = version 1

PiConfigSchema is `z.literal(1)`. Future versions use a `migrations.ts` table. For v0.8.4, no migration needed (no prior versions).

**Decision:** Zod schema with `version: z.literal(1)`. Add migration registry in v0.8.5+ if needed.

## 3. Agent modes are user-extensible

The `modes[]` field in PiConfig is an array of `AgentMode`. v0.8.4 ships `build` and `plan` as defaults. Users can add `review`, `debug`, `commit`, etc.

**Decision:** `modes: AgentMode[]` is a config-time list. `cycleMode()` and `getNextMode()` walk the list. Default is `getDefaultModes()` returning `[build, plan]`.

## 4. Tab cycles agent mode (per Sid's OpenCode request)

Sid said: "when i click tab it switches to plan and build, like opencode you can add more modes if u want". So Tab = cycle, not toggle. Each Tab press advances to the next mode in `modes[]` (wraps).

**Decision:** `Tab` key in any input mode dispatches to `onTab()` callback. App layer calls `cycleMode(current, modes)`. State stored at app level, not in PromptInput (orthogonal to editor state).

## 5. Bordered editor + modal vim coexist

v0.8.4 introduces `EditorFrame` (zentui-style bordered input). The existing modal vim from v0.8.1 lives inside. No refactor of vim state machine.

**Decision:** EditorFrame wraps PromptInput. The accent rail uses `agentColor(agent)` (per-agent color). `copyFriendly` mode hides the rail for cleaner terminal copy.

## 6. Plan mode = read-only tools + bash DESTRUCTIVE_PATTERNS

Adapted from pi-main's plan-mode example. Plan mode restricts tools to `["read", "bash", "grep", "find", "ls", "questionnaire"]` and blocks destructive bash commands via regex (~30 patterns: `rm -rf /`, `curl|sh`, `sudo`, `chmod 777`, `git push`, `npm publish`, etc.).

**Decision:** LlmWorker has `setActiveTools(tools|null)` and `setBashAllowlist(patterns|null)`. The `gateToolCall()` helper in `@pi/subagent/tool-gate` is the single enforcement point.

## 7. Nerd Font detection = heuristic with override

`detectNerdFontsFromEnv()` checks TERM, TERM_PROGRAM, FONT_NAME for known Nerd Font names. `PROMYRA_NERD_FONTS=0/1` overrides detection.

**Decision:** Heuristic detection is opt-out. ASCII fallback always available. Users in non-Nerd-Font terminals still get functional UI (just without the icons).

## 8. zentui-style UI overhaul (NOT port pi-zentui verbatim)

Sid asked for a UI overhaul. We could install `pi-zentui` as a dependency, but it's a separate extension that needs the pi-mono extension API. pi-pro has its own component model. So we built equivalents in our stack (StarshipFooter, EditorFrame, git-status parser, runtime detector).

**Decision:** Build our own components. Don't add a `pi-zentui` dependency. ~33 tests of pure logic + Ink components.

## 9. Live LLM bench still deferred

Same as v0.7.0/v0.8.0/v0.8.1/v0.8.2/v0.8.3. The OpenCode Go API key returns 401 on completion endpoints. Activation requires Sid visiting https://opencode.ai/auth.

**Decision:** Keep bench data as projected (from spec §3). Re-run when Go subscription is active.

## 10. Tab in normal mode: dispatch via `onTab` prop

We don't bake the cycle logic into PromptInput. Instead, the dispatch layer is a single function call. The App layer owns the mode state and calls `cycleMode()` from @pi/config.

**Decision:** PromptInput is a "dumb" component that emits onTab. App layer owns the state. This keeps the test simple (one smoke test, no ink-testing-library fragility).

## 11. CLI wiring kept minimal

We rewrote `pi config` (was v0.2.0 stub). Added `pi mode`, `pi plan`, `pi todos`, `pi ui`. Did NOT add `--config <path>` flag yet (planned but not done — would be 1 line in cli.ts).

**Decision:** Ship what we have. Add `--config` flag in v0.8.5 (low priority).

## 12. No new dependencies

v0.8.4 doesn't add any new npm deps. `@pi/config` depends on `zod` (already in root). All other packages reuse existing deps (ink, react, @types/node, etc.).

**Decision:** Keep dep surface flat. New code lives in existing packages + 1 new package (@pi/config).

## 13. Tests cover the new surface area

Every new feature has TDD coverage:
- @pi/config: 34 tests
- tui-pro additions: 91 new tests
- subagent additions: 15 new tests
- apps/pi-pro additions: 10 new tests

**Decision:** TDD discipline preserved. No untested code in v0.8.4.
