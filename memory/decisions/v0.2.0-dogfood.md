## Issues found

1. **Provider name → env var mapping was wrong** in `getEnvKey()` — fixed mid-session. Need to add unit test for the mapping in v0.2.1.
2. **`getContextUsage()` is not a documented ExtensionAPI method** — my `:context` command casts to find it, but it may not be there. Needs verification with a TTY session.
3. **`/doctor` and other slash commands don't fire in `--print` mode** — only fire in interactive TTY. Can't fully verify without TTY.
4. **Tab shortcut, footer rendering, widget rendering** — also TTY-only. Verified code compiles + tests pass; real visual check needs interactive session.
5. **Session log shows `pi-pro-plan-mode` custom message** but with `display: false` — it shows in the context but not in user-visible output. That's the right behavior for a system prompt.

## v0.2.1 candidates

- Unit test for getEnvKey() provider→env var mapping
- TTY smoke test (script(1) wrapper or playwright on the TUI)
- Add `:mode cycle` arg
- Tighter destructive pattern matching (some safe commands might still get blocked; some unsafe might slip through)
- Persistent memory across sessions (currently memState is in-memory only — `~/.pi/pi-pro/memory.jsonl` is the path, but file write isn't implemented yet)
