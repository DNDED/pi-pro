# v0.2.5 — OpenCode feel + capability foundation

## What changed

### Architecture
- pi-pro-ext refactored from inline code into modules: `commands` (manifest), `mode`, `auth`, `palette`, `header`, `editor-chrome`, `theme-write`, `keybindings-write`, `pi-pro-meta`, `plan-runtime`.
- Single source of truth: `src/commands.ts` defines 15 `PaletteCommand` entries (mode, plan, todos, config, doctor, theme, apikey, memory-add/list/search/clear, palette, help, btw, context). `index.ts` registers them data-driven.

### OpenCode orange theme
- On `session_start`, write orange to `~/.pi/agent/zentui.json`:
  - `colors.editorAccent: "#EC5B2B"`
  - `colors.editorBorder: "#EC5B2B"`
  - `colors.editorPrompt: "#EE7948"`
  - `colors.editorModel: "#EE7948"`
  - `colorSources.editor: "theme"`
- Idempotent: preserves all unrelated zentui.json keys (icons, features, etc.)
- Opt-in via `piPro.theme` in our own `pi.json`; `:theme opencode|default` to switch

### Custom "PI pro" header
- `setHeader` Component rendered above chat
- Layout: `PI pro · v0.2.5 · [+] BUILD · 5.0K/200.0K (3%) · [github] [docs] [/help]`
- 3 OSC 8 hyperlinks (clickable in Ghostty/Kitty/WezTerm/iTerm2/VSCode)
- Falls back to plain text on terminals without OSC 8

### Custom editor chrome
- `setEditorComponent` override (`BorderChromeEditor extends CustomEditor`)
- Braille spinner in top border during agent activity
- Model + thinking level in bottom border
- Hides pi-mono's built-in working indicator to avoid two spinners

### Command palette (Ctrl+P)
- Wrote `~/.pi/agent/keybindings.json` to rebind 3 conflicting actions (`app.model.cycleForward`, `app.session.togglePath`, `app.models.toggleProvider`) to F2
- Triggered `ctx.reload()` after write so the new bindings take effect this session
- Idempotent + backs up existing file to `keybindings.json.bak.<ts>`
- Registered `ctrl+p` shortcut → opens two-step palette (input query → select filtered)
- Fuzzy substring match + suggested commands pinned first
- Also registered as `:palette` slash command

### `:apikey` (replaces dead `:login`)
- pi-mono has built-in `/login` that conflicted with our `/login` (silently blocked)
- Renamed to `:apikey`, dropped the colon alias (pi-mono only parses `/` as the command prefix)
- Same behavior: writes to `~/.pi/agent/auth.json` with mode 0600
- Accepts key as arg or prompts via `ctx.ui.input()`

### Plan widget
- Re-parses on every new plan text (was only once per session in v0.2.4)
- Clears widget when leaving plan mode

### Installed Tier 1 extensions
- `pi-ask-user` (10.7K/mo) — searchable selection UI
- `pi-web-access` (91.3K/mo) — web search / fetch / GitHub / PDF / YouTube
- `pi-mcp-adapter` (99.2K/mo) — MCP proxy
- `pi-lens` (26K/mo) — 37 LSP servers + 32 formatters

## Bug fixes during build

- `parsePlanItems` rejected single-char step text ("A" with `length > 1` check) — fixed to `> 0`
- `setEditorComponent` not in mock UI — added
- Multiple `session_start` handlers — consolidated into one
- Colon aliases don't dispatch in real pi (only `/`) — removed
- pi-tui not directly resolvable from extension — added as dependency
