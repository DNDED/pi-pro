# pi-pro v0.7.0 â€” Memory at Scale Implementation Plan

> **For agentic workers:** TDD-first, bite-sized tasks, frequent commits. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Ship a context manager + memory store + codebase index + /btw that lets pi-pro run long sessions (50+ turns, 100k+ tokens) without losing quality, plus cross-session recall and semantic codebase search. Build on the v0.5.0 Token/Cost Foundation and v0.6.0 Agent Swarm.

**Architecture:** New `packages/embeddings` (provider abstraction), `packages/memory-store` (SQLite + embeddings + hybrid search), `packages/context-manager` (sliding window + extractive + LLM-summarize + adaptive triggers + /btw), `packages/codebase-index` (regex symbols + embeddings + hybrid search). The LLM worker wraps context-manager; the TUI surfaces context budget.

**Tech Stack:** TypeScript, Node 20+, pnpm workspaces, vitest, `better-sqlite3` (pure-JS SQLite), `chokidar`, existing v0.5.0+v0.6.0 packages.

---

1. **`packages/embeddings`** (NEW) â€” provider abstraction + 3 implementations + NullEmbeddings fallback.
