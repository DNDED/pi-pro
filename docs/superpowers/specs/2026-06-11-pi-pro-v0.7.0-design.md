## 4. Components

### 4.1 `packages/embeddings` (NEW)

Pluggable embeddings provider abstraction. Lives at top of dependency stack.

```ts
interface EmbeddingsProvider {
  readonly name: string;
  readonly dim: number;             // 384 for MiniLM, 1024 for Voyage-3, 1536 for OpenAI ada
  embed(text: string): Promise<Float32Array>;
  embedBatch(texts: string[]): Promise<Float32Array[]>;
}
```

Implementations:
- `AnthropicEmbeddings` (Voyage-3 via opencode-go passthrough)
- `OpenAIEmbeddings` (text-embedding-3-small, $0.02/M tokens)
- `OpenCodeGoEmbeddings` (provider's own endpoint)
- `NullEmbeddings` (BM25-only fallback when no API key / no provider)

Default: same provider as the LLM. Override via `PROMYRA_EMBEDDINGS=<provider>`.

### 4.2 `packages/memory-store` (NEW)

| Module | Purpose |
|---|---|
| `src/db.ts` | SQLite schema + migrations; tables for `chunks`, `embeddings`, `projects`, `sources` |
| `src/store.ts` | CRUD: `addChunk(text, embedding, meta)`, `query(text, k)`, `delete(id)` |
| `src/embedder.ts` | Embeds chunks via `packages/embeddings` provider |
| `src/indexer.ts` | Indexes project markdown + codebase chunks on demand. Lazy + cached. |
| `src/query.ts` | Hybrid search: vector cosine + recency boost + source-type filter; falls back to BM25 if no embedding |

**Schema:**
```sql
CREATE TABLE chunks (
  id INTEGER PRIMARY KEY,
  project TEXT,           -- "pi-pro" or null for global
  source TEXT,            -- "user.md" | "decision-log" | "code:src/foo.ts:42"
  role TEXT,              -- "fact" | "preference" | "decision" | "code-symbol"
  text TEXT NOT NULL,
  embedding BLOB NOT NULL,
  ts INTEGER NOT NULL
);
CREATE INDEX idx_chunks_project ON chunks(project);
CREATE VIRTUAL TABLE chunks_vss USING vss0(embedding(384));
```

Without vss: brute-force cosine search in JS, fast for <10k chunks. Add vss when scale demands.

### 4.3 `packages/context-manager` (NEW)

| Module | Purpose |
|---|---|
| `src/budget.ts` | Token budget per turn; tracks accumulated tokens; trigger evaluation |
| `src/window.ts` | Sliding window: evicts oldest non-system messages when budget exceeded |
| `src/extract.ts` | Extractive compression: keep user/assistant text verbatim, drop oldest 50% of tool results first |
| `src/summarize.ts` | LLM-summarization: recursive call to summarize dropped conversation when extractive insufficient |
| `src/triggers.ts` | Adaptive trigger: token threshold (75% soft-warn, 90% hard), turn count (every 20), cost cap (configurable) |
| `src/btw.ts` | `/btw` side question: separate LLM call, doesn't pollute main history, returns answer to user only |
| `src/integration.ts` | Wires context-manager into the LLM worker loop |

### 4.4 `packages/codebase-index` (NEW)

Combines v0.5.0 repo-map with persistent + embedding-indexed symbol table.

| Module | Purpose |
|---|---|
| `src/scanner.ts` | Wraps v0.5.0 repo-map (regex symbol extraction) |
| `src/embedder.ts` | Embeds code chunks (function bodies + symbol signatures + file headers) |
| `src/storage.ts` | Persists to memory-store (source: "code:...") |
| `src/search.ts` | Hybrid query: regex exact + vector semantic + path filter |
| `src/watcher.ts` | chokidar file watcher; incremental re-index on save |

### 4.5 `packages/tui-pro` (extended)

| Component | Purpose |
|---|---|
| `ContextBudget.tsx` (NEW) | Live token bar in Footer: `tok:84k/200k (42%)` with soft-warn at 75% (yellow), hard at 90% (red) |

- `LlmWorker.runWithMemoryContext(question)` — queries memory-store + codebase-index, injects into system prompt

## 5. Data flow

```
User: "Add /healthz"
        ↓
LlmWorker.run(role, ctx)
        ↓
ContextManager.assemble(role, goal) {
  query: memory-store.query(role=builder, k=10) → top-10 relevant chunks
  query: codebase-index.search(goal) → top-5 matching symbols
  assemble: system = static_block + memory_context + codebase_context
  budget: 80k tokens, soft-warn 75%, hard 90%
  compression: not needed (under threshold)
  return OptimizedTurn
}
        ↓
Optimizer.optimize(turn) → cache hints applied
        ↓
provider.complete(messages, opts) → stream tokens
        ↓
ContextManager.recordUsage(role, in_tokens, out_tokens, cache_read, cache_write) {
  budget.total += tokens
  triggers.check() → if any: runExtract(turn) or runLLMSummarize(turn)
  persist(cost.json) to scratchpad
}
        ↓
Loop until done / budget exhausted
```

## 8. Error handling

| Failure | Behavior |
|---|---|
| Embeddings API call fails | Fall back to BM25 (lexical only); log warning |
| SQLite read/write fails | Best-effort; degrade to in-memory cache; surface error to user |
| Compression throws | Skip compression; continue with full context (may exceed budget) |
| LLM summarization fails (timeout, rate limit) | Skip summarization; log; continue with extractive only |
| Memory store has zero chunks on first run | No memory injection; user can `/memory-add` to seed |
| Codebase index fails (e.g. syntax errors) | Skip; fall back to repo-map only |
| `/btw` LLM call fails | Surface error; don't pollute history |
| Embeddings cost exceeds budget | Warn + fall back to BM25 |

## 10. Risks & Mitigations

1. **SQLite + sqlite-vss dep complexity.** Mitigation: ship without vss first; brute-force cosine search for <10k chunks is fast enough. Add vss when scale demands.
2. **Embeddings API cost.** Mitigation: hybrid search (BM25 + vector); cache embeddings for unchanged chunks; budget caps per session.
3. **Compression quality variance.** Mitigation: LLM-summarization gated by quality check (verify key facts preserved); fallback to no compression if quality drops.
4. **Memory store gets huge.** Mitigation: TTL on chunks; user can `/memory-forget`; max chunks per project cap (default 1000).
5. **Cross-session memory leaks sensitive data.** Mitigation: per-project isolation; opt-in to global memory sharing; clear scrub on `/memory-forget`.
6. **Auto-summarization in the middle of a swarm run** (v0.6.0). Mitigation: summarization happens between turns, not during; orchestrator is unaffected.
