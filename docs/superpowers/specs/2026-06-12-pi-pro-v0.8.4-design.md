## 2. Architecture

### 2.1 New package: `@pi/config`

Foundation for all v0.9.0+ work (plugins, LSP, web viewer). The schema:

```ts
// packages/config/src/types.ts
export interface PiConfig {
  version: 1;
  provider: { name: string; model: string; baseUrl?: string };
  agent: { name: string; maxIterations: number; toolBudget: number };
  theme: { name: string; customColors?: Record<string, string> };
  keybindings?: Record<string, string>;
  swarm?: { defaultBudgetUsd: number; defaultRetries: number };
  context?: { embeddings: "openai" | "anthropic" | "opencode-go" | "null"; compression: "off" | "extractive" | "llm" | "hybrid"; memoryQueryK: number };
  ui?: {
    editor: boolean;          // bordered editor with accent rail
    statusLine: boolean;      // Starship-style footer
    copyFriendly: boolean;    // hide accent rail for clean copy
    nerdFonts: boolean;       // Nerd Font icons (auto-detected otherwise)
    icons: Record<string, string>;
    colors: Record<string, string>;
    gitStatusIntervalMs: number;
  };
  modes?: AgentMode[];        // v0.8.4 adds this; extensible
}

export interface AgentMode {
  name: string;               // "build" | "plan" | ...
  label: string;              // "BUILD" | "PLAN" | ...
  color: string;              // theme token
  activeTools: string[];      // tool names allowed
  bashAllowlist?: RegExp[];   // optional: bash patterns allowed
  systemPromptAppend?: string;// optional: injected into LLM system prompt
  readOnly: boolean;          // true: cannot edit/write
}
```

### 2.2 Config file locations

- Primary: `~/.pi/pi.json` (XDG-aware: `$XDG_CONFIG_HOME/pi-pro/pi.json` overrides)
- Override: `--config <path>` CLI flag (transient; doesn't write)
- Schema: `version: 1`; migration table for v1.1+ (none yet)

### 2.3 Precedence (lowest тЖТ highest)

1. Built-in defaults (DEFAULT_CONFIG in `@pi/config`)
2. `~/.pi/pi.json` (or XDG equivalent)
3. `PROMYRA_*` env vars (legacy, v0.5.0+ flags preserved)
4. `--config <path>` flag (transient, read-only override)
5. Inline `set` call (e.g. `pi config set model gpt-4o`) тАФ writes to `~/.pi/pi.json`

### 2.4 Agent mode cycle

**State machine:**
- `AgentMode` is global, parallel to `VimMode` (which is editor-local)
- `agentMode: "build" | "plan"` (extensible via `modes[]` in config)
- Cycle order: build тЖТ plan тЖТ build
- Bindings: `Tab` (global) + `/mode` command + `Ctrl+P` shortcut

**Tab in PromptInput** dispatches to `onTab` prop callback. Caller (App) cycles agent mode. Re-renders show new mode badge.

**Plan mode effects (LlmWorker.setActiveTools + tool gate):**
- Tools restricted to `["read", "bash", "grep", "find", "ls", "questionnaire"]`
- `bash` commands filtered through `bashAllowlist` (DESTRUCTIVE_PATTERNS blocked)
- `before_agent_start` injects `[PLAN MODE ACTIVE]` system message (display: false)
- `mode: "plan"` badge shown in editor frame

### 2.5 zentui-style UI overhaul

Inspired by `pi-zentui` (1,893 downloads/mo). Adapts for pi-pro's stack.

**Components:**
- `StarshipFooter` (replaces `Footer`): git branch + status icons + runtime version + ctx/cache/cost + extension status slots
- `EditorFrame`: bordered input with accent rail; model + provider inside frame; copy-friendly mode (no rail)
- `util/git-status.ts`: parses `git status --porcelain`, formats with Nerd Font icons (`! ? + тЖС тЖУ` etc.) and ASCII fallback
- `util/runtime-detect.ts`: walks cwd for 50+ language markers; `package.json` тЖТ Node + version; `Cargo.toml` тЖТ Rust; `pyproject.toml` тЖТ Python; `go.mod` тЖТ Go; etc.
- `util/nerd-fonts.ts`: heuristic detection (`$TERM`, `fc-list`); ASCII fallback

**Config (in `~/.pi/pi.json`):**
```json
{
  "ui": {
    "editor": true,
    "statusLine": true,
    "copyFriendly": false,
    "nerdFonts": true,
    "icons": { "cwd": "ґ░Э░", "git": " ", "modified": "!" },
    "colors": { "cwd": "bold cyan", "gitBranch": "bold purple" },
    "gitStatusIntervalMs": 5000
  }
}
```

### 2.6 Ex mode extensions

Build on v0.8.2 `:w/:q/:wq/:q!/:clear/:help` and v0.8.3 PromptInput integration. New:

- `:set <key> <value>` тАФ update config key, persist to `~/.pi/pi.json`
- `:set?` тАФ dump current config
- `:unset <key>` тАФ revert to default
- `:r <filename>` тАФ read file at cursor (or buffer-start if cursor at 0)
- `:e <filename>` тАФ replace buffer with file contents
- `:r! <cmd>` тАФ insert command output at cursor
- `:cd <path>` тАФ change cwd (session-scoped)
- `:pwd` тАФ show cwd
- `:mode [name]` тАФ switch agent mode (same as Tab)

## 5. Risks

1. **Git status polling** is slow on huge repos. Cache with 5s TTL; refresh on demand only.
2. **Runtime detection 50+ languages** тАФ 10 file lookups max per render; cached for session.
3. **Nerd Font detection** is heuristic. Fallback always ASCII.
4. **Bordered editor + modal vim** must coordinate rendering. Pre-render border, mount PromptInput inside.
5. **LlmWorker.setActiveTools** must coexist with v0.6.0 swarm tool restrictions. Tool gate = intersection of mode + swarm.
6. **Config migration** is v0.8.4's first schema. No prior versions to migrate. Future bumps use a `migrations.ts` table.

## 7. Decisions (captured at build time)

- Config schema version 1; migrations table to be added in v0.8.5+ if needed
- Agent modes are user-extensible via `modes[]` in config
- Plan mode = read-only; future "review" mode = comment-only; "debug" mode = log-only
- Tab key bound globally; `/mode` + `Ctrl+P` as explicit alternatives
- Bordered editor wraps existing PromptInput (no refactor of vim state machine)
- Git status defaults to 5s TTL; configurable per-cwd
- Nerd Font detection: `$TERM=*nerd*` or `fc-list | grep -i nerd`; else ASCII
- API key never written to config (still in `~/.pi/pi-auth.json` with 0600)
- Live LLM bench still deferred (Go subscription not active)
